# Waitkit GitHub Labels Reference

Use labels only if they exist in the repository.

## Suggested Labels

- `feature`: new package behavior or public capability.
- `bug`: bug fix.
- `documentation`: README, ROADMAP, or API docs.
- `test`: test coverage changes.
- `tooling`: build, lint, TypeScript, Turbo, or pnpm changes.
- `release`: changesets, changelog, package metadata, or publish workflow.
- `harness`: `.claude/**`, `CLAUDE.md`, or agent workflow changes.

If labels are unavailable, create the PR without labels rather than inventing
repository metadata.
